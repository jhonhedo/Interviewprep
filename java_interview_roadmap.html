Strategy

8-week plan · 2–3 hours/day · 5 days/week
Phase 1 (Wk 1–2): Core Java + DSA fundamentals · Phase 2 (Wk 3–4): Spring Boot + SQL · Phase 3 (Wk 5–6): System Design + LLD · Phase 4 (Wk 7–8): Projects + Mock Interviews

Phase 1 · Weeks 1–2 Foundation — Core Java + DSA ▶
Week 1 — Core Java
  • OOP deep dive: polymorphism, abstraction, encapsulation (real-world use, not textbook)
  • Collections: HashMap internals, ArrayList vs LinkedList, TreeMap, HashSet
  • Multithreading: Thread lifecycle, synchronized, volatile, wait/notify
  • Java 8+: Streams, Optional, Lambda, Functional interfaces, CompletableFuture
  • Exception handling best practices, try-with-resources
Week 2 — DSA Fundamentals
  • Arrays, Strings, Two-pointer, Sliding window
  • HashMap/HashSet patterns (frequency count, grouping)
  • Stack and Queue (monotonic stack, BFS, DFS)
  • Binary Search (standard + on answer space)
  • Recursion & Backtracking basics
Phase 2 · Weeks 3–4 Spring Boot + SQL Mastery ▶
Week 3 — Spring Boot & Microservices
  • Spring IOC, DI, Bean lifecycle, @Component vs @Service vs @Repository
  • REST API design: request mapping, validation, exception handling
  • Spring Security: JWT auth, filters, OAuth2 basics
  • JPA/Hibernate: N+1 problem, lazy vs eager, query optimization
  • Microservices: Feign, Eureka, API Gateway, circuit breaker
Week 4 — SQL & Database
  • Joins, subqueries, window functions (ROW_NUMBER, RANK, PARTITION BY)
  • Indexes: types, when to use, composite index order, covering index
  • Query optimization: EXPLAIN ANALYZE, slow query log
  • Transactions: ACID, isolation levels, deadlocks
  • Normalization, denormalization trade-offs
Phase 3 · Weeks 5–6 System Design + LLD ▶
Week 5 — System Design
  • Scalability pillars: horizontal scaling, load balancing, caching
  • Design URL shortener, rate limiter, notification system
  • Database choices: SQL vs NoSQL, when to use Redis/Kafka
  • CAP theorem, eventual consistency, sharding
  • API design: REST vs gRPC, versioning, rate limiting
Week 6 — Low-Level Design
  • SOLID principles with Java code examples
  • Design patterns: Singleton, Factory, Builder, Strategy, Observer
  • Design a parking lot, library system, food delivery model
  • UML class diagrams (basic)
  • Code quality: naming, SRP in practice
Phase 4 · Weeks 7–8 Projects + Mock Interviews + HR ▶
Week 7 — Project polish + Resume
  • Add pagination, caching, and JWT to your main project
  • Write a README with architecture diagram
  • Resume: 4 bullet points per project with metrics
  • LeetCode: 3 medium problems/day, timed (30 min each)
Week 8 — Full Mock + HR Prep
  • 2 full mock interviews (technical + system design)
  • Behavioral: STAR stories for top 10 HR questions
  • Revise Top 50 questions (see Top 50 tab)
  • Apply to 10+ companies — shortlist by target band
A OOP & Java Internals ▶

90% of mid-level interviews test OOP with scenario-based questions, not definitions.

What to study
  • Polymorphism: runtime vs compile-time, method hiding vs overriding
  • Abstract class vs Interface (Java 8+ default methods)
  • Immutability: String pool, how to design immutable class
  • equals/hashCode contract — broken HashMap behavior
  • final, static, transient — real usage
Interview questions
Why is String immutable in Java? What's the benefit?
Can you override a static method? What happens if you try?
Design an immutable class with a mutable field (e.g., a list).
If I put a mutable object as a HashMap key and change it — what happens?
B Collections & Generics ▶

HashMap internals are the #1 asked topic. Know load factor, rehashing, TreeNode at threshold 8.

What to study
  • HashMap: internal hashing, put/get, collision resolution, Java 8 treeification
  • ConcurrentHashMap vs Hashtable vs Collections.synchronizedMap
  • ArrayList vs LinkedList — when to use each (be specific)
  • Comparable vs Comparator
  • Generics: bounded wildcards (? extends T, ? super T), type erasure
Interview questions
How does HashMap handle collisions internally in Java 8?
When would you pick LinkedHashMap over HashMap?
What's the difference between fail-fast and fail-safe iterators?
C Multithreading & Concurrency ▶

Asked heavily at product companies. Focus on practical patterns, not theory.

What to study
  • Thread vs Runnable vs Callable vs Future
  • ExecutorService, ThreadPoolExecutor, ForkJoinPool
  • synchronized vs Lock, ReentrantLock, ReadWriteLock
  • volatile — visibility guarantee, why not atomicity
  • Deadlock: cause, detection, prevention strategies
  • CompletableFuture: thenApply, thenCompose, allOf
Interview questions
Implement a thread-safe Singleton using double-checked locking.
What is a race condition? Show a code example and fix it.
Difference between wait() and sleep()? Why is wait() inside synchronized?
How would you implement a producer-consumer without using BlockingQueue?
D Java 8–17 Features ▶

Streams are a must. If you can't write stream pipelines fluently, you look junior.

What to study
  • Stream: map, filter, reduce, collect, flatMap, distinct, sorted, limit
  • Collectors: groupingBy, partitioningBy, toMap, joining
  • Optional: correct usage, avoid null checks
  • Functional interfaces: Predicate, Function, Consumer, Supplier, BiFunction
  • Method references: types (static, instance, constructor)
  • Record classes, var, sealed classes (Java 14–17)
Interview questions
Group a list of employees by department and find the max salary in each group using streams.
What is the difference between map() and flatMap()?
When would you NOT use Optional? What are its pitfalls?
E Spring Core & Boot Internals ▶

Interviewers test "what happens under the hood," not just @Autowired usage.

What to study
  • IOC container, BeanFactory vs ApplicationContext
  • Bean scopes: Singleton, Prototype, Request, Session
  • @Component, @Service, @Repository, @Controller — differences
  • Auto-configuration: how @SpringBootApplication works
  • @Transactional: propagation levels, isolation, rollback rules
  • AOP: @Aspect, @Around, @Before — practical logging/timing use case
Interview questions
What's the difference between @Component and @Bean?
Explain @Transactional propagation: REQUIRED vs REQUIRES_NEW.
How would you handle circular dependency in Spring?
What happens if you call a @Transactional method from within the same class?
F REST APIs, Security & JPA ▶

Almost every real project uses JPA + REST + JWT. This is the bread and butter of the interview.

What to study
  • REST best practices: HTTP verbs, status codes, idempotency
  • @ControllerAdvice, @ExceptionHandler for global error handling
  • Spring Security: filter chain, JWT validation, custom UserDetailsService
  • JPA: N+1 problem (fix with JOIN FETCH or @EntityGraph), lazy loading pitfalls
  • Pagination, sorting with Spring Data (Pageable)
  • @Query with JPQL vs native SQL
Interview questions
Explain the N+1 query problem. How do you diagnose and fix it?
How does JWT authentication work in a Spring Boot filter chain?
What's the difference between @OneToMany with EAGER vs LAZY?
How would you design a global exception handler for your REST API?
G Microservices Patterns ▶

If the JD says "microservices," you'll be asked these. Know patterns by name and code.

What to study
  • Service discovery: Eureka, Consul (how registration/heartbeat works)
  • API Gateway: Spring Cloud Gateway — routing, filters, rate limiting
  • Feign client: declarative REST calls, fallback
  • Circuit breaker: Resilience4j — states (CLOSED/OPEN/HALF_OPEN)
  • Distributed tracing: Micrometer + Zipkin/Jaeger
  • Event-driven: Kafka basics — producer, consumer, consumer groups, offset management
Interview questions
How do you handle partial failure when calling 3 downstream services?
What is a circuit breaker? Walk through the state transitions.
How do you maintain data consistency across microservices without distributed transactions?
What is the SAGA pattern? Choreography vs orchestration?
H Core Patterns to Master ▶

At 3 YoE, you won't face hard DP. Focus on medium-level patterns — these cover 80% of questions asked.

Must-know patterns (in priority order)
  • Two pointers — sorted arrays, pair sum, palindrome checks
  • Sliding window — max subarray, longest substring without repeat
  • HashMap patterns — frequency count, anagram grouping, subarray sum
  • Binary search — on sorted array + on answer space (capacity, days)
  • Stack patterns — monotonic stack (next greater element, histogram area)
  • BFS/DFS — graphs, trees, island counting, shortest path
  • Linked list — reverse, detect cycle (Floyd), merge sorted lists
  • Recursion + Memoization — basic DP (knapsack, coin change, LCS)
  • Tree traversals — level order, zigzag, lowest common ancestor
  • Heap (PriorityQueue) — top K elements, merge K sorted lists
LeetCode target list (solve 80–100 problems)
  • Two Sum, 3Sum, Container with Most Water, Trapping Rain Water
  • Longest Substring Without Repeating Characters, Minimum Window Substring
  • Binary Search, Search in Rotated Array, Find Peak Element
  • Valid Parentheses, Daily Temperatures, Next Greater Element
  • Number of Islands, Word Ladder, Clone Graph
  • LRU Cache (LinkedHashMap or custom), Median of Data Stream
  • Coin Change, Longest Common Subsequence, House Robber
Daily DSA routine
  • 1 easy (warm-up) + 1–2 mediums — 45–60 min total
  • After solving, check top solution on LeetCode Discuss
  • Maintain a "patterns journal" — one line per problem with the pattern used
I SQL Queries & Window Functions ▶

Window functions separate seniors from juniors in SQL rounds. Master these.

What to study
  • All JOIN types — INNER, LEFT, RIGHT, FULL OUTER, CROSS, SELF
  • Subqueries vs CTEs (WITH clause) — readability + performance
  • Window functions: ROW_NUMBER, RANK, DENSE_RANK, LAG, LEAD, SUM/AVG OVER PARTITION
  • GROUP BY with HAVING — filter on aggregates
  • NULL handling: COALESCE, NULLIF, IS NULL vs = NULL
Interview questions
Find the second-highest salary in each department.
Find employees who earn more than their manager.
Get the running total of sales by date using window functions.
Find duplicate rows and delete all but one.
J Indexes, Optimization & Transactions ▶

Performance tuning questions are extremely common at product companies where query speed matters.

What to study
  • Index types: B-tree, hash, composite, covering index
  • When NOT to index (high write, low cardinality, small tables)
  • EXPLAIN ANALYZE — reading query plan, identifying full table scans
  • ACID: Atomicity, Consistency, Isolation, Durability — real examples
  • Isolation levels: READ UNCOMMITTED, READ COMMITTED, REPEATABLE READ, SERIALIZABLE
  • Deadlock detection and resolution strategies
  • Indexing composite columns: column order matters (leftmost prefix rule)
Interview questions
A query on a 10M row table is slow. What is your step-by-step debug approach?
You have an index on (A, B, C). Does a query filtering on B and C use the index?
What is a phantom read? Which isolation level prevents it?
When would you prefer a NoSQL database over a relational one?
K Core Building Blocks ▶

At 3 YoE, you need to demonstrate structured thinking, not perfect answers. Learn the vocabulary.

Essential concepts
  • Load balancing: round-robin, least connections, consistent hashing
  • Caching: Redis, cache-aside, write-through, TTL, eviction (LRU)
  • CDN: static asset delivery, edge caching
  • Message queues: Kafka vs RabbitMQ — use cases, ordering guarantees
  • Database sharding: horizontal partitioning strategies
  • SQL vs NoSQL: when each — Cassandra for time-series, MongoDB for docs
  • CAP theorem: trade-offs in distributed systems
L Must-Practice Designs ▶
Design these end-to-end (30 min each, practice out loud)
  • URL shortener (hashing, redirect, analytics, DB choice)
  • Rate limiter (token bucket, sliding window, Redis implementation)
  • Notification system (email/SMS/push, fan-out, delivery guarantees)
  • E-commerce product search (indexing, Elasticsearch, filters)
  • Chat system (WebSocket, message storage, delivery receipts)
  • Payment processing (idempotency, saga, retries)
Framework for every design question
  • 1. Clarify requirements (2 min) — scale, users, features, constraints
  • 2. Estimate scale — requests/sec, storage, bandwidth
  • 3. High-level diagram — services, DB, cache, queue
  • 4. Deep dive on 1–2 components the interviewer picks
  • 5. Discuss trade-offs and bottlenecks

M SOLID Principles in Java ▶

Know how to apply, not just define. Show code examples in Java.

SOLID — with Java violations and fixes
  • SRP: One reason to change — split UserService from EmailService
  • OCP: Open for extension, closed for modification — use Strategy pattern
  • LSP: Subclass must not break parent behavior — Rectangle vs Square trap
  • ISP: Prefer small interfaces — split Flyable, Swimmable from Animal
  • DIP: Depend on abstractions — inject Repository interface, not concrete class
Design patterns to master (with Java code)
  • Singleton — thread-safe lazy init (enum approach)
  • Factory + Abstract Factory — object creation without tight coupling
  • Builder — complex object construction (used everywhere in Spring)
  • Strategy — replace if-else chains with polymorphism
  • Observer — event listeners, pub-sub
  • Decorator — wrapping behavior (InputStream hierarchy)
  • Template Method — define skeleton, subclass fills steps
LLD Design Practice problems
  • Parking lot (levels, vehicle types, ticketing, fees)
  • Library management system (book, member, issue, return)
  • Chess game (board, pieces, rules, player)
  • Food delivery (restaurant, rider, order, status FSM)
  • ATM machine (PIN, balance, withdraw, card states)
N Projects that stand out ▶

Rule: 1 strong project beats 5 basic CRUD apps. Add depth: async processing, caching, auth, pagination, monitoring.

Build one of these (with full stack)
  • E-commerce backend — product catalog, cart, order, inventory, payment (Stripe/Razorpay simulation)
  • Task management API — real-time updates via WebSocket, role-based access, audit logs
  • URL shortener — custom alias, analytics, Redis caching, rate limiting per user
  • Job portal microservices — 3+ services (user, job, notification), Kafka between them, Docker Compose
Must-haves in every project
  • JWT authentication + role-based authorization
  • Global exception handling (proper HTTP status codes)
  • Pagination, filtering, sorting via Spring Data
  • Redis caching on at least one endpoint
  • Unit tests (JUnit 5 + Mockito) — min 60% coverage
  • README with architecture diagram + setup instructions
  • Deployed on Railway/Render or Dockerized
O Resume optimization ▶
Resume rules for 8–15 LPA roles
  • 1 page only. No photo, no "Objective" section.
  • Each bullet: Action verb + What you did + Result/metric
  • Bad: "Worked on backend APIs" → Good: "Built 12 REST APIs for order management; reduced avg latency by 35% with Redis caching"
  • Skills section: Java, Spring Boot, Hibernate, MySQL, Redis, Kafka, Docker, Git — no "MS Word"
  • Projects: Name, tech stack in brackets, 3 bullet points with impact
  • Use ATS-friendly format (no tables, columns, graphics)
Behavioral prep — STAR format
  • Tell me about yourself — 90-second pitch: current role → what you built → why you're looking
  • Biggest challenge — use a real technical problem, explain your debugging process
  • Conflict with a colleague — focus on resolution, not blame
  • Greatest achievement — tie it to business impact, not just code
  • Why this company — research 2–3 specific things about their tech stack

Mock interview strategy

Week 4 — Start mocks early

  • Use Pramp or interviewing.io for free peer mocks
  • Record yourself explaining — watch for clarity and filler words
  • Do at least 1 mock/week from Week 4 onward

Technical round framework

  • DSA: Think out loud, start brute force, optimize step by step
  • System Design: Clarify → Estimate → Diagram → Deep dive → Trade-offs
  • Ask clarifying questions — shows seniority

Negotiation tips

  • Never give a number first — ask for their range
  • Get competing offers if possible — real leverage
  • Negotiate base, bonus, stock separately
  • Know your BATNA before any offer call
